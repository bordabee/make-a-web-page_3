Lesson 1: The Basics of the Web and HTML

Introduction to the Web

The World Wide Web is a collection of HTML documents.
HTML stands for HyperText Markup Language, this is the basis for the majority of web pages.
HTML is what connects most types of documents together through hyperlinks. HTML can be made up of text content (what you see), markup (arrangement of content), references to other documents, and links to other pages.

Computers are Stupid

Programmers need to write in a language a computer understands; this way a computer can follow the instructions the programmer wrote.
Therefore, in order to be precised, it is imperative to use the right syntax.

HTML Attributes

Anchor tags: adds a link to a text or word. When clicked, it directs you to another page.
Image tags: represents an image.
Whitespace: text on multiple lines using the break tag (void tag) or the paragraph tag (not a void tag).
Inline/Block: the inline element makes the content remain in the line, while a block tag creates an invisible box. Examples of inline tags include the anchor, span, break, image, and strong tags. While the paragraph and div tags are types of block tags.

Document Structure

The basic structure of an HTML document includes:

  • a DOCTYPE in the first line,
  • an opening HTML tag on the second line,
  • an opening and closing head tags, which will include title tags,
  • an opening an closing body tags, which will include the page's content,
  • and a closing html tag.

Lesson 3: Adding CSS Style and HTML Structure

Understanding CSS

CSS stands for Cascading Style Sheet and is the code that styles HTML.
Cascading means that most specific rule gets applied. This inheritance is the mechanism by which properties are applied not only to a specified element, but also to its descendants. In general, descendant elements inherit text-related properties, but box-related properties are not inherited. Inheritance
Inheritance enhances faster-loading of web pages by users and enables clients to save money on bandwidth and development costs.
There are three ways of including CSS to webpage:

  • Writing CSS in the head element of an HTML structure
  • Linking HTML to a separate CSS file
  • Writing CSS inline within the HTML
Benefits of Avoiding Repetition

Faster Websites: fewer repeated styles in CSS result in smaller file sizes, and thus faster downloading of those resources. Maintainable Stylesheets: when making additions to an existing site, it is easier to reuse styles and making extensions based on existing rules.

Box Sizing

The core of the box is defined by the width and height of an element, which may be determined by the display property, by the contents of the element, or by specified width and height properties. Padding and border expand the dimensions of the box outward from the elementâ€™s width and height. Any margin specified will follow the border.
Each part of the box model corresponds to a CSS property.
By default the box model is additive; thus to determine the actual size of a box one needs to take into account padding, borders, and margins for all four sides of the box.
The width not only includes the width property value, but also the size of the left and right padding, left and right borders, and left and right margins.
The content-box value is the default value, leaving the box model as an additive design. If the box-sizing property is not used, this will be the default value for all elements.

Box Positioning

Because the div tag is a block element, by default it takes the entire width of a page. In order to override this action, it is recommended to add display: flex; to the CSS.
The Flexbox Layout Module aims at providing a more efficient way to lay out, align and distribute space among items in a container, even when their size is unknown and/or dynamic.
The main idea behind the flex layout is to give the container the ability to alter its items' width, height, and order to best fill the available space, and mostly to accommodate to all kind of display devices and screen sizes. A flex container expands items to fill available free space, or shrinks them to prevent overflow.
